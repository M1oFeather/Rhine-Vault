# 安全与约束边界

## 路径安全

- 客户端只能传 workspace_id、node_id、entry_id；
- 不接受任意路径；
- 所有路径由服务端 resolver 生成；
- resolve 后必须确认仍位于 vault 根目录；
- 禁止 `..`、绝对路径、驱动器前缀和符号链接逃逸。

## 写入安全

- 正式文件仅能通过审核服务写入；
- MCP Agent 只能提交草稿；
- 原子替换；
- 文件 hash；
- revision 乐观锁；
- 审计日志不可被普通业务接口修改。

## 工作区隔离

每个 repository 查询必须同时带 workspace 条件。禁止先按 node_id 全局查询后再进行应用层过滤。

## 提示注入防护

知识节点内容属于数据，不属于系统指令。提供给 Agent 时应明确包裹来源和边界，节点正文中的“忽略规则”等内容不能获得更高指令权限。

## 离线边界

默认：

- 不自动下载模型；
- 不调用云端嵌入；
- 所有网络功能显式配置；
- 启动时输出当前是否处于严格离线模式。

## 数据恢复

至少支持：

- 全量重新扫描 Markdown；
- 重建 SQLite 登记；
- 重建全文索引；
- 重建向量索引；
- 报告无法解析的节点，而不是静默跳过。
